2023辦護照熱潮

7月 23日 2023

為什麼大家都在辦護照？

隨著疫情已經逐漸退去，經歷了近三年的疫情生活，台灣在2023年的暑假掀起了出國旅遊的熱潮，人們都想在這個暑假把之前沒有玩到的玩回來。也因為太久沒出國，護照都已經過期了，所以才會有這麼多人擠在現在辦護照。

辦護照要準備什麼？

先附上申請護照資訊連結，護照分成兩種辦法：

  • 現場辦
    直接拿著你的資料到現場辦理
  • 網路預約
    先在網路預約系統約個日期與時段，再拿你的資料到現場的網路預約處辦理
    網路預約申辦系統連結

需要準備的資料有：

  1. 簡式護照資料表 下載連結
  2. 舊護照 (如果你的舊護照還沒過期的話)
  3. 身分證或居留證
  4. 符合護照格式的大頭照

此處僅列出18歲以上所需要帶的資料，其它詳細規定請看申請換發普通護照說明書

網路預約辦理日細節

因為我本人是網路預約的，2023六月底的時候去系統上面看，最近的只能預約到七月中的時段，非常搶手，建議大家早點預約辦理。
到了預約日當天，我就直接到台北的外交部領事事務局辦理，一開始還走錯棟，視力比較不好一點，這裡先附上地圖，希望大家不要跟我一樣走錯棟。

Linux Fundamental

對於Linux一竅不通的我很想找機會來好好學習它，現在放假了，當然要來好好學一下，透過上次介紹的HTB Academic平台來學習。

Linux Fundamental這個Module有配給你一台一天可以用兩個小時的雲端虛擬機，它可以直接ssh進練習的目標機器，不需要再設定VPN，但是呢!它整個課程只能使用兩小時真的是不太夠，我都還沒到課程的一半它就結束了，只好甘願用自己的虛擬機來連線。

教學方式

這個課程有分成多個章節，可以讓你循序漸進的熟悉linux指令，我自己的linux基礎並不是很深，所以對才剛接觸linux的我來說，這個課程鰻花時間的。每一章都有一些題目，有分兩種，實作和簡答。

  • 實作題：
    要你連進目標機器拿到題目要的資訊
  • 簡答題：
    單純回答題目而已，通常google就會有答案了

Vpn連線

補充一下，我是用自己電腦上面的虛擬機來練進目標機器的vpn的，使用的作業系統是Debian，先到題目附近下載vpn檔案之後就在自己的terminal裡面打上
sudo openvpn 'vpn檔案路徑'
然後再點’spawn the target’得到目標機器的ip位置，就可以執行
ssh htb-student@ip.address
記得虛擬機的網路設定要設定成橋接模式vpn才連得通喔!

HTB系列介紹

7月 13日 2023

什麼是HTB?

HTB Hack The Box，是一個可以讓你學習各種駭客技巧，讓你變成一個資安大佬的地方。有分成四種地方

  • HTB Academic
  • HTB Labs
  • HTB Business
  • HTB CTF

這些功能就是顧名思義而已，有針對學生的Academic和針對企業員工訓練的Business版本，還有像是CTF的競賽區，Labs就是可以進行各種操作的地方。HTB就是給你一個目標，你要去達成它，這些目標都是對一台虛擬機作一些事情或是得到機器裡面的資料，其實就是CTF的形式，因為最近學習慾爆棚，一直對CTF很嚮往的我終於要來試試看這個酷東西了。
我最近主要是在Academic還有Labs的地方活動，這些網站內以類似遊戲的感覺來鋪陳，讓你覺得好像每完成一個任務，就離破關更進一步，是蠻讓人想一直解下去的，解完也會有很大的成就感。

Academic

裡面有數不清的課程給你選(它的課程叫做module)，每一個課程都用看起來很酷的封面包裝，讓你真的很想買下去! 沒錯，課程是需要購買的!，用站內的一種貨幣，一開始他會給你70顆綠方塊(? ，然後這些綠方塊可以透過課程中的實作部分免費得到，但是我看到現在好像也沒有課程是可以賺綠方塊的，頂多就是你花多少綠方塊拿回多少這樣，好像真的要花錢才能買到更進階的課程。

除了自己選課程，你還可以選擇他們預先幫你挑好的課程包(叫做Path)，每個課程包都會是有個主題，有點你上完這些課，你就可以精通這個技術的感覺，學起來感覺也會蠻有方向的。

目前我有買了個學Linux基礎的課程，希望可以在裡面獲得許多札實的硬知識! (升大三了再不熟Linux就完蛋)!

之後應該會持續紀錄各種在HTB裡學到的新東西，期待之後的文章!

Floating Point Binary的運算方法

7月 11日 2023

Floating Point binary 的運算方法

表示方法 (IEEE Floating-Point Format)


  • single (32bits)

    • Exponent: 8 bits
    • Fraction (尾數): 23 bits
    • Bias: 127 bits
    • S: 正數或負數

    特性

    1. Exponent 00000000 和 11111111為保留數:
      • 00000000: 代表正負0或denormalized number
      • 11111111: 代表正負無限或NaN
    2. 最小/最大值:
      • 最小:
        • Exponent = 00000001 $\to 2^{1-127}=2^{-126}$
        • Fraction = 00000000 $\to significand=1.0$
        • $\pm 1.0 \times 2^{-126} \approx \pm 1.2 \times 10^{-38}$
      • 最大:
        • Exponent = 11111110 $\to 2^{254-127}=2^{127}$
        • Fraction = 11111111 $\to significand \approx 2.0$
        • $\pm 2.0 \times 2^{127} \approx \pm 3.4 \times 10^{38}$
    3. Precision:
      只跟Fraction有關 (23 bits)
      $23 \times log_{10}2 \approx 23 \times 0.3 \approx 6$ decimal digits of precision

  • double (64bits)

    • Exponent: 11 bits
    • Fraction (尾數): 52 bits
    • Bias: 1203 bits
    • S: 正數或負數

    特性

    1. Exponent 000…0 和 111…1為保留數:
      • 000…0: 代表正負0或denormalized number
      • 111…1: 代表正負無限或NaN
    2. 最小/最大值:
      • 最小:
        • Exponent = 000…01 $\to 2^{1-1023}=2^{-1022}$
        • Fraction = 000…00 $\to significand=1.0$
        • $\pm 1.0 \times 2^{-1022} \approx \pm 2.2 \times 10^{-308}$
      • 最大:
        • Exponent = 111…10 $\to 2^{2046-1203}=2^{1023}$
        • Fraction = 111…11 $\to significand \approx 2.0$
        • $\pm 2.0 \times 2^{1023} \approx \pm 1.8 \times 10^{308}$
    3. Precision:
      只跟Fraction有關 (52 bits)
      $52 \times log_{10}2 \approx 52 \times 0.3 \approx 16$ decimal digits of precision

Canonical Cover

7月 11日 2023

Canonical Cover

Definition

The simpliest set of $F$, i.e. the functional dependencies without Extraneous attributes, denotes by $F_c$

Extraneous Attributes

  • Is the redundant attributes in each functional dependencies in $F^+$, i.e. $F^+$ doesn’t change without extraneous attributes.

  • Split to two part:

    • Remove attributes on left hand side
      ex:
      $F={AB\to C, A\to D, D\to C}$
      $AB\to C$ have extaneous attribute $B$, because $A\to C$ is in $F^+$, $B$ can be removed without changing $F^+$
      簡化過的$F$ : $F^{\prime}={A\to C, A\to D, D\to C}$
      Simpliest set of $F$ : $F={A\to CD, D\to C}$
    • Remove attibutes on right hand side
      ex:
      $F={AB\to CD, A\to C}$
      $AB\to CD$ have extaneous attribute $C$, because when we replace $AB\to CD$ with $AB\to D$, $C$ is in $(AB)^+$ too, ($(AB)^+=ABDC$), so $C$ can be remove without changing $F^+$
      簡化過的$F$ :
      $F^{\prime}={AB\to D, A\to C}$
      Simpliest set of $F$ :
      $F_c={AB\to D, A\to C}$

Testing Decomposition for BCNF

7月 11日 2023

Testing Decomposition for BCNF

  • Must check all $R_i$ in the decomposition of $R$
  • Just to check the FDs that violate BCNF in each $R_i$

Start Testing

  1. In each $R_i$ , find every set of $\alpha\ in\ R_i$, check $\alpha^+$ either includes
    • no attribute of $R_i - \alpha$
    • includes all attributes of $R_i$
      If all set of $\alpha$ satisfied these, this $R_i$ is in BCNF, otherwise, if some FDs violates these rules, and $R_i$ violates BCNF
  2. Find the dependencies violating BCNF, using $\alpha\to(\alpha^+-\alpha)\cap R_i$
  3. Decompose the violating relation in to two relation
    • $\alpha \cup \beta$
    • $R_i-(\beta-\alpha)$

Normal Form (NF)

7月 11日 2023

Normal form (NF):

  • 1NF
  • 2NF
  • 3NF

1NF:

  • The domains of all attributes are single value and atomic (non
    divisible
  • 一個column裡的type要一樣
  • 每個column的name不能一樣

2NF:

  • 必須是1NF
  • No partial dependencies，only full dependencies
    • non-prime attribute
      不是primary key的attribute
    • Partial dependencies:
      (A is candidate key)
      B is functionally dependent on A ,
      and on any proper subset of A
    • Full dependencies:
      (A is candidate key)
      B is functionally dependent on A ,
      but not on any proper subset of A

3NF:

  • 必須是1NF和2NF
  • No transitive dependencies
    • Transitive dependencies: $\alpha \to \beta,\ \alpha$ is not primary/composite key, and $\beta$ is not primary/compsite key
  • 有FD($\alpha\to\beta\ in\ F^+$)滿足下列至少一項
    • $\alpha \to \beta$ is trivial
    • $\alpha$ is a superkey for R
    • Each attribute in $\beta-\alpha$ is contained in a candidate key for R.
