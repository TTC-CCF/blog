Floating Point Binary的運算方法

7月 11日 2023

Floating Point binary 的運算方法

表示方法 (IEEE Floating-Point Format)


  • single (32bits)

    • Exponent: 8 bits
    • Fraction (尾數): 23 bits
    • Bias: 127 bits
    • S: 正數或負數

    特性

    1. Exponent 00000000 和 11111111為保留數:
      • 00000000: 代表正負0或denormalized number
      • 11111111: 代表正負無限或NaN
    2. 最小/最大值:
      • 最小:
        • Exponent = 00000001 $\to 2^{1-127}=2^{-126}$
        • Fraction = 00000000 $\to significand=1.0$
        • $\pm 1.0 \times 2^{-126} \approx \pm 1.2 \times 10^{-38}$
      • 最大:
        • Exponent = 11111110 $\to 2^{254-127}=2^{127}$
        • Fraction = 11111111 $\to significand \approx 2.0$
        • $\pm 2.0 \times 2^{127} \approx \pm 3.4 \times 10^{38}$
    3. Precision:
      只跟Fraction有關 (23 bits)
      $23 \times log_{10}2 \approx 23 \times 0.3 \approx 6$ decimal digits of precision

  • double (64bits)

    • Exponent: 11 bits
    • Fraction (尾數): 52 bits
    • Bias: 1203 bits
    • S: 正數或負數

    特性

    1. Exponent 000…0 和 111…1為保留數:
      • 000…0: 代表正負0或denormalized number
      • 111…1: 代表正負無限或NaN
    2. 最小/最大值:
      • 最小:
        • Exponent = 000…01 $\to 2^{1-1023}=2^{-1022}$
        • Fraction = 000…00 $\to significand=1.0$
        • $\pm 1.0 \times 2^{-1022} \approx \pm 2.2 \times 10^{-308}$
      • 最大:
        • Exponent = 111…10 $\to 2^{2046-1203}=2^{1023}$
        • Fraction = 111…11 $\to significand \approx 2.0$
        • $\pm 2.0 \times 2^{1023} \approx \pm 1.8 \times 10^{308}$
    3. Precision:
      只跟Fraction有關 (52 bits)
      $52 \times log_{10}2 \approx 52 \times 0.3 \approx 16$ decimal digits of precision

Canonical Cover

7月 11日 2023

Canonical Cover

Definition

The simpliest set of $F$, i.e. the functional dependencies without Extraneous attributes, denotes by $F_c$

Extraneous Attributes

  • Is the redundant attributes in each functional dependencies in $F^+$, i.e. $F^+$ doesn’t change without extraneous attributes.

  • Split to two part:

    • Remove attributes on left hand side
      ex:
      $F={AB\to C, A\to D, D\to C}$
      $AB\to C$ have extaneous attribute $B$, because $A\to C$ is in $F^+$, $B$ can be removed without changing $F^+$
      簡化過的$F$ : $F^{\prime}={A\to C, A\to D, D\to C}$
      Simpliest set of $F$ : $F={A\to CD, D\to C}$
    • Remove attibutes on right hand side
      ex:
      $F={AB\to CD, A\to C}$
      $AB\to CD$ have extaneous attribute $C$, because when we replace $AB\to CD$ with $AB\to D$, $C$ is in $(AB)^+$ too, ($(AB)^+=ABDC$), so $C$ can be remove without changing $F^+$
      簡化過的$F$ :
      $F^{\prime}={AB\to D, A\to C}$
      Simpliest set of $F$ :
      $F_c={AB\to D, A\to C}$

Testing Decomposition for BCNF

Testing Decomposition for BCNF

  • Must check all $R_i$ in the decomposition of $R$
  • Just to check the FDs that violate BCNF in each $R_i$

Start Testing

  1. In each $R_i$ , find every set of $\alpha\ in\ R_i$, check $\alpha^+$ either includes
    • no attribute of $R_i - \alpha$
    • includes all attributes of $R_i$
      If all set of $\alpha$ satisfied these, this $R_i$ is in BCNF, otherwise, if some FDs violates these rules, and $R_i$ violates BCNF
  2. Find the dependencies violating BCNF, using $\alpha\to(\alpha^+-\alpha)\cap R_i$
  3. Decompose the violating relation in to two relation
    • $\alpha \cup \beta$
    • $R_i-(\beta-\alpha)$

Normal Form (NF)

7月 11日 2023

Normal form (NF):

  • 1NF
  • 2NF
  • 3NF

1NF:

  • The domains of all attributes are single value and atomic (non
    divisible
  • 一個column裡的type要一樣
  • 每個column的name不能一樣

2NF:

  • 必須是1NF
  • No partial dependencies，only full dependencies
    • non-prime attribute
      不是primary key的attribute
    • Partial dependencies:
      (A is candidate key)
      B is functionally dependent on A ,
      and on any proper subset of A
    • Full dependencies:
      (A is candidate key)
      B is functionally dependent on A ,
      but not on any proper subset of A

3NF:

  • 必須是1NF和2NF
  • No transitive dependencies
    • Transitive dependencies: $\alpha \to \beta,\ \alpha$ is not primary/composite key, and $\beta$ is not primary/compsite key
  • 有FD($\alpha\to\beta\ in\ F^+$)滿足下列至少一項
    • $\alpha \to \beta$ is trivial
    • $\alpha$ is a superkey for R
    • Each attribute in $\beta-\alpha$ is contained in a candidate key for R.

Multivalued Dependencies

Multivalued Dependencies

  • Definition:
    Denoted by $\alpha \twoheadrightarrow \beta$
    $\alpha$ uniquly identify the set of $\beta$
  • Theory
    • if $\alpha\to\beta$, then $\alpha \twoheadrightarrow \beta$
      That is, every functional dependency is also a multivalued dependency
    • closure $D^+$ of $D$
      Is the set of all all functional and multivalued dependencies logically implied by $D$.
  • 4NF
    • if for all multivalued dependencies in $D^+$ of the form $\alpha \twoheadrightarrow \beta$, where $\alpha \subseteq R$ and $\beta \subseteq R$, at least one of the following hold:
      • $\alpha \twoheadrightarrow \beta$ is trivial (i.e., $\beta \subseteq \alpha$ or $\alpha \cup \beta = R$)
      • $\alpha$ is a superkey for schema $R$

Quicksort Analysis

7月 11日 2023

Quicksort Analysis

Struckture:

  • Recursive function
    • Partition function

Best Case and Worse case:

  • Best Case (for completely balanced subarray)
    $O(n\log n)$
    $T(n)=2T(n/2) + \Theta(n)=\Theta(n\log n)$
  • Worse Case (for completely unbalanced subarray)
    $O(n^2)$
    $T(n)=T(n-1)+T(0)+\Theta(n) = \Theta(n^2)$

Randomized Version of Quicksort

  • Purpose:
    Preventing the algorithm run into worse case.
  • Analysis:
    $Claim 1:$ “partition” function will be called at most n times.
    $Claim 2:$ The amount of work is n times of calls + the times of comparisons performed in all calls to “partition” function (denoted by $X$). So the entire execution is $O(n+X)$
    \begin{equation}E[X]=\sum_{i=1}^{n-1}\sum_{j=i+1}^{n}P(A[i]\ compare\ with\ A[j])\end{equation}
    $P(A[i]\ compare\ with\ A[j])\
    =P(A[i]\ is\ the\ first\ pivot\ in\ A_{ij})\
    \quad+P(A[j]\ is\ the\ first\ pivot\ in\ A_{ij})\
    =\frac1{j-i+1}+\frac1{j-i+1}\
    =\frac2{j-i+1}$
    \begin{equation}\begin{split}E[X]&=\sum_{i=1}^{n-1}\sum_{j=i+1}^{n}\frac2{j-i+1}\
    &=\sum_{i=1}^{n-1}\sum_{k=1}^{n-i}\frac2{k+1}\
    &<\sum_{i=1}^{n-1}\sum_{k=1}^n\frac2{k}\
    &=\sum_{i=1}^{n-1}O(\log n)\
    &=O(n\log n)\end{split}\end{equation}

Master Method

7月 11日 2023

Master Method

\begin{equation}T(n) = aT(n/b)+f(n)\end{equation}
compare $n^{\log_b a}$vs.$f(n)$
If $n^{\log_b a}$grows polynomially faster than $f(n)$
$\to\ Case\ 1$
If $f(n)$ grows polynomially faster than $n^{\log_b a}$
$\to\ Case\ 3$
If $f(n)=n^{\log_b a}$
$\to\ Case\ 2$

Case 1:

$f(n)=O(n^{\log_b {a-e}})\ ,e>0$
solution:
$T(n)=\Theta(n^{\log_b a})$

Case 2:

$f(n)=O(n^{\log_b a}lg^k n)$, $k\geqslant 0$
solution:
$T(n)=\Theta(n^{\log_b a}lg^{k+1}n)$

Case 3:

$f(n)=\Omega(n^{\log_b a+e})\ , e>0$
(因為$f(n)$ is polynomially greater than $n^{\log_b a}$所以要檢查regularity condition $af(n/b)\leq cf(n)\ ,where\ c<1$，但如果$f(n)$是polynomial可以不用檢查)
solution:
$T(n)=\Theta(f(n))$
