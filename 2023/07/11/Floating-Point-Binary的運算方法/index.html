Floating Point binary 的運算方法

表示方法 (IEEE Floating-Point Format)


  • single (32bits)

    • Exponent: 8 bits
    • Fraction (尾數): 23 bits
    • Bias: 127 bits
    • S: 正數或負數

    特性

    1. Exponent 00000000 和 11111111為保留數:
      • 00000000: 代表正負0或denormalized number
      • 11111111: 代表正負無限或NaN
    2. 最小/最大值:
      • 最小:
        • Exponent = 00000001 $\to 2^{1-127}=2^{-126}$
        • Fraction = 00000000 $\to significand=1.0$
        • $\pm 1.0 \times 2^{-126} \approx \pm 1.2 \times 10^{-38}$
      • 最大:
        • Exponent = 11111110 $\to 2^{254-127}=2^{127}$
        • Fraction = 11111111 $\to significand \approx 2.0$
        • $\pm 2.0 \times 2^{127} \approx \pm 3.4 \times 10^{38}$
    3. Precision:
      只跟Fraction有關 (23 bits)
      $23 \times log_{10}2 \approx 23 \times 0.3 \approx 6$ decimal digits of precision

  • double (64bits)

    • Exponent: 11 bits
    • Fraction (尾數): 52 bits
    • Bias: 1203 bits
    • S: 正數或負數

    特性

    1. Exponent 000…0 和 111…1為保留數:
      • 000…0: 代表正負0或denormalized number
      • 111…1: 代表正負無限或NaN
    2. 最小/最大值:
      • 最小:
        • Exponent = 000…01 $\to 2^{1-1023}=2^{-1022}$
        • Fraction = 000…00 $\to significand=1.0$
        • $\pm 1.0 \times 2^{-1022} \approx \pm 2.2 \times 10^{-308}$
      • 最大:
        • Exponent = 111…10 $\to 2^{2046-1203}=2^{1023}$
        • Fraction = 111…11 $\to significand \approx 2.0$
        • $\pm 2.0 \times 2^{1023} \approx \pm 1.8 \times 10^{308}$
    3. Precision:
      只跟Fraction有關 (52 bits)
      $52 \times log_{10}2 \approx 52 \times 0.3 \approx 16$ decimal digits of precision
