HTTP connection process

  • Non-persistent HTTP
  • Persistent HTTP

Non-persistent HTTP

  1. Client initiate TCP connection to HTTP server
  2. HTTP server waiting for TCP connection and send “accept” to client
  3. Client send request into TCP connection socket
  4. HTTP server receives request message, forms response message and send it into TCP connection socket
  5. Client receives response message
  6. One object downloaded
  7. Repeat step 1 to 5 until objects are downloaded

Persistent HTTP

Step 1 to 5 is the same as non-persistent connection, but persistent connection requires only once initiate TCP connection, and send request until objects are downloaded
cutting respond time in half
